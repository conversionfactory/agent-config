Explain $ARGUMENTS in a way that builds understanding:

1. **Analogy first**: Compare it to something from everyday life
2. **Visual diagram**: Create an ASCII or mermaid diagram showing the flow/structure
3. **Step-by-step walkthrough**: Walk through how it works
4. **Key insights**: What's clever or important about this approach?
5. **Gotchas**: Common mistakes or misconceptions

Make the explanation memorable and actionable.
